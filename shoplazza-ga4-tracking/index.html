Shoplazza GA4 Tracking Starter Guide

This is the practical setup path for Shoplazza operators: make sure the campaign visit lands correctly, then verify the same session and ecommerce events inside GA4 before scale.

Open the GA4 checklist Open the free builder

First proof point

The session appears in GA4 Realtime with the source, medium, and campaign labels you intended.

Second proof point

The event path still looks believable after product view, cart, checkout, and purchase steps where supported.

Common issue

The landing page reads correctly but later checkout or purchase tracking breaks because the platform path changes.

Starter workflow

  1. Generate one clean campaign URL from the homepage builder.
  2. Open GA4 Realtime before clicking the URL.
  3. Click the link and confirm the correct Shoplazza page opens with the parameters still attached.
  4. Check the source, medium, and campaign values in Realtime.
  5. Walk through the product and checkout flow where your platform setup supports it.

What to verify on Shoplazza

  • The platform integration or custom code path is not firing duplicate GA4 events.
  • Localization, redirects, or apps are not stripping your campaign parameters.
  • Purchase attribution remains connected to the original session instead of dropping later in the flow.
  • The event chain stays believable after the first page view.

Good practical rule

Keep the first version of the setup boring and testable. A simple repeatable verification loop is more useful than a fancy tracking setup you cannot explain when it breaks.
