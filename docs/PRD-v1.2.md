**当前状态：** 已上线，迭代优化

## 1. 执行摘要

### 产品愿景
为猫主人提供一个快速、可信、易浏览的植物安全查询和发现平台，让用户能更安心地装饰居家环境。

### 一句话产品描述
输入任意植物名，用户即可快速判断它是否对猫安全，同时浏览安全植物、危险植物、替代植物与真实用户故事，并能直接获取购买建议。

### 目标用户和使用场景
- 英语语种国家的猫主人
- 在买花、养花、布置家居前想快速确认安全性
- 想浏览 cat-safe houseplants、low-light plants、pet-friendly home decor 的用户

### 关键差异化
现有站点已有内容卡片和故事内容基础。本次迭代重点补齐搜索入口、分类页、替代推荐、收藏与分享，并前置商业化导流入口。让用户从"看内容"变成"查问题、做决策、去购买"。

### 成功定义
- 每日 UV 稳定增长
- 搜索使用率提升，搜索零命中率下降
- 页面停留时长提升
- 收藏 / 分享 / 二次访问提升

## 2. 问题与机会

### 当前问题
现在首页以内容卡片为主，用户有明确"查毒性"需求时路径过长。同时，用户查完毒性后缺乏明确的"下一步（如购买替代品）"引导，流量未形成商业闭环。

### 用户痛点
- 不确定植物是否对猫有毒
- 现有信息分散，缺少一站式查询
- 想买植物时没有替代建议

### 机会
利用结构化数据和清晰的判定标准（SSOT），截获 Google 关于植物安全性的精准搜索流量（SEO），并通过替代推荐切入家居植物购买链路。

---

## 3. 迭代目标

### 本期目标
把现有内容站改造成 **内容 + 查询 + 分类 + 转化** 的混合型产品。

### 业务目标
- 提升自然搜索流量（重点优化 SEO 与结构化数据）
- 提升站内浏览深度
- 为未来的 Affiliate 商业化做前置铺垫

### 用户目标
- 快速查到植物是否猫安全
- 找到适合自己家的替代植物（并能直接购买）

---

## 4. 用户需求

### 核心用户故事

1. **作为猫主人**，我希望搜索植物名后立即看到安全结论，以便决定是否购买。
2. **作为植物爱好者**，我希望看到安全替代品，以便继续装饰家里。
3. **[新增]** **作为搜索未命中的用户**，我希望有一个简单的入口提报缺失的植物，以便网站未来能覆盖。

## 5. MVP 范围

### Must Have
- **植物安全搜索**：首页加搜索框，支持别名
- **搜索兜底 (Zero-Result Handling)**：搜不到时提供"提报该植物"入口，并推荐安全替代品
- **植物详情页标准化**：统一展示判定结论（SSOT）、护理信息、风险提示、替代推荐，并预留商业转化按钮（Purchase Link）
- **分类页**：Safe / Toxic / Low-light / Easy-care
- **基础收藏与分享**：本地收藏，生成社交媒体适配的分享链接

### Should Have
- **今日植物模块**：固定展示每日植物
- **FAQ / Safety Note**：明确免责声明与安全提示

### Won't Have for Now
- 图片识别、复杂个性化推荐、社区评论、完整社交账户体系

## 6. 产品方案与逻辑架构

```
用户搜索 Query
    ↓
词库精确/模糊匹配
    ↓
命中 → 植物详情页
未命中 → Zero-Result 无结果页
            ↓
        推荐热门安全植物
            ↓
        记录 Query & 用户提报入口

植物详情页
    ↓
安全状态判定 SSOT
    ↓
Safe → 展示养护指南
Risky/Toxic → 展示毒性症状与警告
                    ↓
                推荐安全替代植物 Alternative
                    ↓
                预留 Affiliate Link 转化入口
```

### 页面核心改造

**首页**：顶部导航 + 主搜索框 + 今日植物 + 快捷分类 + 热门风险植物

**详情页**：名称 + 图片 + 严格的安全标签（Enum）+ 简短解释 + 毒性症状（如有）+ 替代植物 + 购买导流 CTA + 收藏/分享。全页需植入 JSON-LD Schema.org 以优化 SEO。

**分类页**：承接精准场景流量（如 Pet-safe small apartment plants）

## 8. 技术要求

### 架构建议
保持现有 Web 架构（Next.js + Supabase），不做重构，专注于模块组件化和数据治理。

### 数据模型最少字段 (SSOT 标准)

| 字段名称 | 类型 | 业务定义与说明 |
|:---|:---|:---|
| plant_id | UUID | 主键 |
| common_name | String | 通用名称，用于主展示 |
| scientific_name | String | 学名，用于精确匹配 |
| aliases | Array/JSON | 别名词库，用于提升搜索命中率 |
| safety_status | Enum | 严格枚举：safe, toxic_to_cats, toxic_to_dogs_and_cats，禁止自由文本 |
| toxicity_symptoms | String | 结构化毒性症状描述（如 Vomiting, Drooling） |
| risk_level | Enum | low, medium, high |
| care_difficulty | Enum | easy, medium, hard |
| light_level | Enum | low, bright_indirect, direct |
| is_featured_today | Boolean | 用于低成本实现"今日植物"卡片拉取 |
| affiliate_url | URL | 商业化预留字段，跳转外部购买链接 |
| alternatives | Array | 关联的替代植物 ID 列表 |

### 接口建议

- `GET /api/plants?query=` - 需处理模糊匹配与别名
- `GET /api/plants/:slug`
- `GET /api/categories/:slug`
- `POST /api/search-misses` - 记录未命中查询语料
