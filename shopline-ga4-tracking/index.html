SHOPLINE GA4 Tracking Starter Guide

This page keeps the setup path simple for independent-store operators using SHOPLINE: verify the landing visit first, then confirm the source labels and ecommerce events inside GA4.

Open the GA4 checklist Open the free builder

First signal

GA4 Realtime sees the session with the same source, medium, and campaign values you intended.

Second signal

The store journey still produces believable ecommerce events after product view and checkout steps.

Main risk

Platform integrations, custom scripts, or checkout behavior can change what survives between landing and purchase.

Starter workflow

  1. Generate one clean campaign URL from the homepage builder.
  2. Open GA4 Realtime and click that URL yourself.
  3. Confirm the correct SHOPLINE page opens and the query string is preserved.
  4. Check source, medium, and campaign inside Realtime.
  5. Walk through product view and checkout steps where your platform setup allows it.

What to verify on SHOPLINE

  • The platform integration or custom code path is not duplicating GA4.
  • Redirects or localization behavior do not strip the campaign parameters.
  • The checkout flow preserves attribution instead of making the purchase look direct or empty.
  • The event chain remains believable beyond the first page view.

Good practical rule

Keep the naming convention stable and the verification routine boring. When the simple path is proven, then layer on more detail only if the reporting question demands it.
